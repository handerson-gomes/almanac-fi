# 042 — Model capabilities and selection

## Story

As a user, I can select a compatible provider/model for a run, workflow, feature, or default.

## Requirements

Implement model descriptors, capability requirements, selection precedence, persistence of preferences, compatibility diagnostics, settings UI, and fallback behavior.

## Acceptance criteria

Selection follows documented precedence; incompatible models cannot run a requirement; actual model is recorded when known; no hard-coded feature model requirement without justification.

## Dependencies

010, 041

## Verification

Test precedence, fallback, and incompatible-model paths.
